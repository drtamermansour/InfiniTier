# CLI Reference

All flags accepted by `run_pipeline.sh` and the underlying step scripts.

## `run_pipeline.sh`

The main entry point — wraps reference indexing, `remap_manifest.py`, and
`qc_filter.py`.

### Required

| Flag | Description |
|---|---|
| `-i / --manifest` | Path to the Illumina manifest CSV |
| `-r / --reference` | Path to the target reference genome FASTA |
| `-a / --assembly` | Assembly name used to label outputs (e.g. `equCab3` → columns like `Chr_equCab3`) |

### General

| Flag | Default | Description |
|---|---|---|
| `-o / --output-dir` | `./output` | Output directory |
| `-t / --threads` | `4` | Threads for minimap2 |
| `--keep-temp` | off | Retain intermediate FASTA/SAM files after `remap_manifest.py` completes |
| `--resume` | off | Skip remapping if the remapped CSV already exists |

### QC filter knobs (forwarded to `qc_filter.py`)

Grouped per `--help`:

**Filter strictness**

| Flag | Default | Description |
|---|---|---|
| `--min-anchor` | `topseq` | Minimum anchor evidence: `dual`=`topseq_n_probe` only; `topseq`=+`topseq_only`; `probe`=+`probe_only` |
| `--tie-policy` | `resolved` | Minimum tie resolution: `unique`; `resolved`=unique+`*_resolved`; `avoid_scaffolds`=resolved+`scaffold_resolved` |
| `--min-refalt-confidence` | `moderate` | Ref/Alt confidence: `high`=`NM_match`+`NM_validated`; `moderate`=+`NM_N/A`+`NM_tied`; `low`=+`NM_only`+`NM_unmatch`+`NM_corrected` |

**Thresholds** (pass `off` to disable)

| Flag | Default | Description |
|---|---|---|
| `--min-mapq-topseq` | `30` | Minimum MAPQ for TopGenomicSeq alignments (0–60, or `off`); `probe_only` markers exempt (their MAPQ is `NaN`) |
| `--min-mapq-probe` | `off` | Minimum MAPQ for probe alignments; `topseq_only` markers exempt |
| `--max-coord-delta` | `off` | Remove markers where `\|probe_cigar_pos − topseq_cigar_pos\| > N`; `topseq_only` and `probe_only` (CoordDelta=−1) pass through |

**Include/exclude** (excluded by default)

| Flag | Default | Description |
|---|---|---|
| `--include-indels` | off | Include indel markers in outputs |
| `--include-polymorphic` | off | Include markers at polymorphic positions |
| `--include-ambiguous-snps` | off | Include ambiguous (A/T or C/G) SNPs |

**Operational**

| Flag | Default | Description |
|---|---|---|
| `--preset` | none | Tune the strictness + threshold + include/exclude flags as a bundle: `strict` / `default` / `permissive`. Individual flags passed alongside `--preset` override whatever the preset set. See the table below. |

### What each `--preset` sets

| Preset | `--min-anchor` | `--tie-policy` | `--min-refalt-confidence` | `--min-mapq-topseq` | `--min-mapq-probe` | `--max-coord-delta` | `--include-indels` | `--include-polymorphic` | `--include-ambiguous-snps` |
|---|---|---|---|---|---|---|---|---|---|
| `strict` | `dual` | `unique` | `high` | `30` | `20` | `1` | off | off | off |
| `default` | `topseq` | `resolved` | `moderate` | `30` | `off` | `off` | off | off | off |
| `permissive` | `probe` | `avoid_scaffolds` | `low` | `off` | `off` | `off` | **on** | **on** | **on** |

Omitting `--preset` is equivalent to `--preset default` — the values in that row are simply the flag defaults. Any individual flag you pass on the command line overrides the corresponding cell in the preset row.

For HPC use:

```bash
bash submit_slurm.sh -i <manifest.csv> -r <reference.fa> -a <assembly> -o results/ -t 64
```

## `scripts/remap_manifest.py` (step 2 directly)

Run only the alignment + coordinate-resolution step.

| Flag | Required | Description |
|---|---|---|
| `-i / --manifest` | yes | Input manifest CSV |
| `-r / --reference` | yes | Reference genome FASTA |
| `-o / --output` | yes | Output remapped CSV path |
| `-a / --assembly` | no, default `new_assembly` | Assembly name used in output column names |
| `--threads` | no, default `4` | Threads for minimap2 |
| `--temp-dir` | no, default = output dir | Where intermediate FASTA/SAM files go |
| `--resume` | off | Reuse SAM files in `--temp-dir` instead of re-aligning |

## `scripts/qc_filter.py` (step 3 directly)

Run only the quality-control filter cascade.

| Flag | Required | Description |
|---|---|---|
| `-i / --input` | yes | Remapped CSV from `remap_manifest.py` |
| `-r / --reference` | yes | Reference genome FASTA |
| `-v / --vcf-contigs` | yes | VCF contig header file (auto-generated by `run_pipeline.sh`) |
| `-a / --assembly` | no, default `new_assembly` | Must match what `remap_manifest.py` was run with |
| `-o / --output-dir` | no, default `.` | Output directory |
| `--min-anchor` | `topseq` | See above |
| `--tie-policy` | `resolved` | See above |
| `--min-refalt-confidence` | `moderate` | See above |
| `--min-mapq-topseq` | `30` | 0–60 or `off` |
| `--min-mapq-probe` | `off` | 0–60 or `off` |
| `--max-coord-delta` | `off` | See above |
| `--include-indels` | off | See above |
| `--include-polymorphic` | off | See above |
| `--include-ambiguous-snps` | off | See above |
| `--preset` | none | `strict` / `default` / `permissive` — tunes the flags above as a set |
| `--temp-dir` | default = output dir | For intermediate files |
| `--prefix` | derived from input filename | Prefix for output file names |

## `scripts/benchmark_compare.py`

Compare a remapped manifest against a ground-truth (e.g., the same array's
EquCab3-native v2 manifest). See [docs/benchmarking.md](benchmarking.md) for
how to interpret results.

| Flag | Required | Description |
|---|---|---|
| `--manifest` | yes | Ground-truth manifest CSV |
| `--remapped` | yes | The pipeline's remapped CSV |
| `--assembly` | no | Assembly label (e.g. `equCab3`). **Auto-detected** from the remapped CSV header when omitted. Pass explicitly only if the CSV contains multiple assemblies (the script will error out with a clear "ambiguous" message in that case). |
| `--reference` | no | Reference FASTA — enables the explanatory verdict layer (manifest_strand_wrong / pipeline_unmapped / pipeline_wrong_chr / pipeline_wrong_locus / etc.) |
| `--traced` | no | Path to `{prefix}_remapped_{assembly}_traced.csv` from `qc_filter.py` — enables the **QC filtration impact** report section. Use the trace from the same preset/assembly combo as the `--remapped` CSV; otherwise the `why_filtered` column will be misleading. |
| `--output-dir` | `./benchmark_out` | Where to write the per-marker TSV and report |
| `--baseline` | none | Path to a prior `benchmark_<ts>.tsv` — adds a "what changed" diff section |
| `--flank-len` | `20` | Number of bases of flanking context to check |

## `scripts/benchmark_cigar_vs_probe.py`

Three-way per-marker accuracy comparison: probe-CIGAR coordinate vs
TopSeq-CIGAR coordinate vs the final chosen coordinate.

| Flag | Required | Description |
|---|---|---|
| `--manifest` | yes | Ground-truth manifest CSV |
| `--remapped` | yes | Remapped CSV |
| `--assembly` | no | Assembly label. Auto-detected from the remapped CSV header when omitted (same rules as `benchmark_compare.py`). |
| `--output-dir` | `./benchmark_out` | Where to write the timestamped `benchmark_cigar_vs_probe_{ts}_report.txt` (timestamp format `YYYY-MM-DD_HH-MM-SS`, matching `benchmark_compare.py`) |

## Pre-pipeline scaffold tools

See [docs/scaffold_filtering.md](scaffold_filtering.md) for the full workflow.

- `scripts/scaffold_haplotype_analyzer.py` — characterise unplaced scaffolds.
- `scripts/filter_scaffold_haplotypes.py` — pick alt-haplotype candidates by threshold.
- `scripts/exclude_alt_haplotypes.py` — build a cleaned reference FASTA.
